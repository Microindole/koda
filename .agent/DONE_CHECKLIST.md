# 完成检查清单（DoD）

提交前必须逐项确认。

## 1. 功能正确性

- [ ] 主流程可运行。
- [ ] 异常/空数据路径有处理。
- [ ] 配置变更与代码行为一致。

## 2. 架构一致性

- [ ] 改动符合路线图方向（感知/状态/表现分层）。
- [ ] 没有把临时逻辑硬编码到页面层。
- [ ] 公共逻辑已抽离为可复用模块。

## 3. 质量与可维护性

- [ ] 新增代码可读，命名清晰。
- [ ] 未完成项已加 `TODO`。
- [ ] 已知未修复问题已加 `FIXME`。
- [ ] 无无效注释、无废弃代码残留。

## 4. 测试与验证

- [ ] 相关测试通过（若有）。
- [ ] 至少做过一次手动验证（关键路径）。
- [ ] 若未验证，已在交付说明中写明原因和风险。

## 5. 性能与合规（按 roadmap）

- [ ] 未引入不必要后台常驻任务。
- [ ] 传感器监听生命周期可释放。
- [ ] 未引入与“本地优先/隐私优先”冲突的设计。

## 6. 交付质量

- [ ] 交付说明包含文件清单与改动原因。
- [ ] 风险、后续工作、依赖项写清楚。
- [ ] 输出语言与风格符合 `.agent/AGENTS.md`。
